Part I


    风险代码：


        NSString *string = array[1][@"key"];


    问题：1.array不一定是NSArray。

         2.array不一定有两个以上元素。

         3.array[1]不一定是个NSDictionary。

         4.array[1][@"key"]不一定是个NSString。

    结论：1 ~ 3可能能造成崩溃，4可能造成逻辑错误甚至崩溃。



    安全代码：


    NSString *string = nil;

    if ([array isKindOfClass:[NSArray class]] &&

        array.count > 1)

    {

        NSDictionary *dictionary = array[1];

        if ([dictionary isKindOfClass:[NSDictionary class]]) {

            id value = dictionary[@"key"];

            if ([value isKindOfClass:[NSString class]]) {

                string = value;

            }

        }

    }


    虽然确保了string的正确性，但是增加了编写成本，也提高了阅读成本。


Part II


    基于以上问题，采取扩展NSArray和NSDictionary的方法，使用起来和原生写法几乎没有区别。


    首先，需要 #import "NSArray+Extension.h"


    对于NSArray，

    

       

    只需在“array”与“[1]”之间加入“.stringAtIndex”，即 NSString *string = array.stringAtIndex[1](此处有代码提示)，

    就可以获取到想要的类型NSString，对于NSNumber，可以用 NSNumber *number = array.numberAtIndex[index]。


    优势：1.不用考虑array是不是个NSArray，如果是，那么就走正确逻辑，如果不是，会返回nil。

         2.不用考虑array有多少个元素，如果index正确，那么就走正确逻辑，如果越界，会返回nil。

         3.不用考虑拿到的类型，如果array.stringAtIndex[1]拿到的是个NSString，那么返回，如果不是，会返回nil。


    回到Demo，NSString *string = array[1][@"key"]，现在可以通过array.dictionaryAtIndex[1][@"key"]拿到@"key"

    对应的object了，但是，还是不能解决问题4。下面就是对NSDictionary的扩展，用法类似于NSArray。


    首先 #import "NSDictionary+Extension.h"


    对于NSDictionary，

    

        

    同NSArray一样，在“dictionary”与“[@"key"]”之间加入想要的类型，如“.stringForKey”，注意，是 ForKey ，不是 AtIndex 。


    另外，还有一点要注意的是NSDictionary的泛型，


    如声明一个NSDictionary<NSString *, NSArray<MyObject *> *> *dictionary;


    case 1:


        NSString *string = dictionary.stringForKey[@1];⚠️Incompatible pointer types sending 'NSNumber *' to parameter of type 'NSString *'


        1.关于key。传key是NSNumber的话就会报警告

        2.关于value。虽然声明了dictionary的value是NSArray<MyObject *>类型，但是该方法取的是NSString，不会触发编译器警告


    case 2:


        value是指定泛型容器，如NSArray<MyObject *>，通过arrayForKey是拿不到泛型的，举例


        1.NSArray *array = dictionary.arrayForKey[@"key"]; // 编译器无警告

        2.NSArray<NSString *> *array = dictionary.arrayForKey[@"key"]; // 编译器无警告，拿不到泛型类型

        3.NSArray *array = dictionary[@"key"]; // 编译器无警告

        4.NSArray<NSString *> *array = dictionary[@"key"]; // 编译器有警告，可以拿到泛型类型


        问题2原因：arrayForKey返回的是个NSArray *，并非NSArray<ObjectType> *，也不能返回NSArray<ObjectType> *，

        具体原因不在此展开。NSArray也同样存在该问题。


Part III


    目前NSArray和NSDictionary实现了NSString、NSNumber、NSArray、NSDictionary的安全取值，如果想要实现更多的类型，可以通过宏扩展。


    扩展NSArray的宏如下：


    NSArrayExtensionDeclare(OneClass, objectName)

    NSArrayExtensionSynthesize(OneClass, objectName)


    实现的方法：


    OneClass *object = array.objectNameAtIndex[1];


    扩展NSDictionary的宏如下：


    NSDictionaryExtensionDeclare(OneClass, objectName)

    NSDictionaryExtensionSynthesize(OneClass, objectName)


    实现的方法：


    OneClass *object = array.objectNameForKey[@"key"];


    如NSData，在.h文件中加入如下宏


    NSArrayExtensionDeclare(NSData, data)

    NSDictionaryExtensionDeclare(NSData, data)


    在.m文件中加入如下宏

    NSArrayExtensionSynthesize(NSData, data)

    NSDictionaryExtensionSynthesize(NSData, data)


    然后就可以愉快的玩耍NSData了


    NSData *data = dictionary.dataForKey[@"key"];

    NSData *data = array.dataAtIndex[1];


    同样支持Custom Class，如MyObject，


    NSArrayExtensionDeclare(MyObject, myObject)

    NSDictionaryExtensionDeclare(MyObject, myObject)


    NSArrayExtensionSynthesize(MyObject, myObject)

    NSDictionaryExtensionSynthesize(MyObject, myObject)


    MyObject *myObject = array.myObjectAtIndex[1];

    MyObject *myObject = dictionary.myObjectForKey[@"key"];
